## Problem

`useBatchTokenData` calls `setTokenData(new Map(accumulated))` on every microtask flush. Each resolved token creates a new Map reference, causing ALL consumers to re-render — even those whose specific token data hasn't changed yet.

## Plan

1. **Debounce the flush** — batch all token resolutions within a single React render cycle using a short timeout (e.g. 50ms) instead of `queueMicrotask`
2. **Or switch to a Zustand store with per-mint selectors** — each consumer subscribes to only its mint key, avoiding unnecessary re-renders
3. Simplest option: accumulate all results and flush once after all promises settle, rather than flushing per-resolution
