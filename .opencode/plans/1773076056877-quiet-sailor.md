# Plan: Restructure Liquidity Chart Data with LiquidityShape

## Overview

Reorganize liquidity chart data into a cohesive `LiquidityShape` structure that groups all position metadata, bin distribution, and token totals together. This improves data organization and makes the component API cleaner.

## Current Issues

- Multiple individual props passed to `LiquidityChart` (chartBins, currentActiveId, currentPrice, minPrice, maxPrice, maxLiquidity)
- Related data scattered across different function calls
- No unified structure for position liquidity information
- Token totals not available to chart component (though calculated separately in PositionCard)

## Proposed Solution

### 1. Define `LiquidityShape` Interface

**File:** `src/utils/positions/calculations.ts`

```typescript
export interface LiquidityShape {
  positionAddress: string
  pairAddress: string
  binRange: {
    minBinId: number
    maxBinId: number
    totalBins: number
  }
  binDistribution: ChartBinData[]
  tokenTotals: {
    tokenX: number
    tokenY: number
  }
}
```

### 2. Create `generateLiquidityShape()` Function

Create a new function that:

- Takes position data, pair address, and token decimals
- Generates bin distribution (reusing existing `generateLiquidityChartData`)

- Returns complete `LiquidityShape` object

```typescript
export function generateLiquidityShape(
  positionData: any,
```

## Implementation Notes

### Token Totals Calculation

Token totals are computed by reducing over `binDistribution`:

```typescript
const tokenTotals = binDistribution.reduce(
  (acc, bin) => ({
    tokenX: acc.tokenX + bin.tokenXAmount,
    tokenY: acc.tokenY + bin.tokenYAmount,
  }),
  { tokenX: 0, tokenY: 0 },
)
```

### Max Liquidity Calculation

In `LiquidityChart`, max liquidity can be derived from:

- Option 1: `tokenTotals.tokenX + tokenTotals.tokenY` (total position value)
- Option 2: `Math.max(...binDistribution.map(b => b.tokenXAmount + b.tokenYAmount))` (peak bin)
- **Recommendation**: Use Option 2 for consistency with current behavior (peak bin drives chart scale)

### Bin Range for Price Display

Price range (min/max) should still be calculated separately in `LiquidityChart` since it depends on `pricePerToken` values from bins, which are already in `ChartBinData`.
