# Plan: Display Meteora DLMM Positions in UI

## Overview

Add UI components to display PositionInfo data that's already being fetched from the meteora-dlmm SDK.

## Implementation Steps

### 1. Create Utility Functions for Data Formatting

**File:** `src/app/positions/utils/formatters.ts`

Create functions to:

- `formatTokenAmount(amount: bigint, decimals: number)` - Format token amounts with proper decimals
- `formatPriceRange(lower: bigint, upper: bigint, binStep: number)` - Display price range
- `formatTimestamp(timestamp: number)` - Human-readable time
- `shortenPublicKey(key: string)` - Truncate wallet addresses
- `formatFees(feeX: bigint, feeY: bigint)` - Display unclaimed fees

### 2. Create Empty State Component

**File:** `src/app/positions/components/EmptyState.tsx`

Simple component showing "No positions found" message when Map is empty.

### 3. Create Position Card Component

**File:** `src/app/positions/components/PositionCard.tsx`

Display each position with:

- Token pair display (token symbols and amounts)
- Price range (lower - upper bin IDs)
- Total X and Y token amounts
- Unclaimed fees (if present)
- Expand/collapse functionality
- Uniwind styling matching existing theme

### 4. Create Positions List Screen

**File:** `src/app/positions/index.tsx`

Main component that:

- Receives `Map<string, PositionInfo>` as prop
- Converts Map to array for rendering
- Renders ScrollView with PositionCards
- Shows EmptyState when no positions
- Handles loading states (optional)

### 5. Integrate into Main App

**File:** `src/app/index.tsx`

Modify to:

- Replace current welcome screen with positions display
- Pass positions state to PositionsList component
- Keep wallet connection UI (move to header area)
- Show loading/error states appropriately

## Styling Guidelines

Use existing Uniwind theme colors:

- `bg-app-bg` - Background
- `bg-app-card` - Card backgrounds
- `text-app-text` - Primary text
- `text-app-text-secondary` - Secondary text
- `text-app-text-muted` - Muted text
- `text-app-primary` - Primary accent

Card structure:

```tsx
<View className="bg-app-card rounded-2xl p-4 mb-3 shadow-sm border border-app-border">{/* Content */}</View>
```

## Key Design Decisions

1. **Simple card layout** - Show summary view first, expandable for details
2. **ScrollView for list** - Simple and sufficient for current use case
3. **Reuse existing theme** - Match current color scheme from index.tsx
4. **Handle edge cases** - Empty state, loading state, error state
5. **Mobile-first** - Full width cards, proper touch targets
