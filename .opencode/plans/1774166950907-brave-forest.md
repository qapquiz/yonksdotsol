# Code Review: UPNL Refactor Change

## Summary

This change refactors how Unrealized PnL (UPNL) is fetched and displayed:

- **Before**: Calculate UPNL locally using initial deposits from Helius API + current position values
- **After**: Use `metcomet` SDK's `getUpnlPerPosition` function which provides UPNL data directly

## Changes Overview

| File                                        | Change                                                 |
| ------------------------------------------- | ------------------------------------------------------ |
| `package.json`, `bun.lock`                  | Upgraded `metcomet` from `^0.1.2` â†’ `^0.2.2`           |

| `src/components/positions/PositionCard.tsx` | Removed local UPNL calculation, accept `upnlData` prop |
| `src/config/cache.ts`                       | Renamed `INITIAL_DEPOSITS` â†’ `UPNL_PER_POSITION`       |
| `src/hooks/positions/useUpnlPerPosition.ts` | **New** hook using SDK                                 |
| Deleted hooks/utilities                     | Removed old hooks that calculated UPNL locally         |

## Bug Analysis

### âś… No Bugs Found - The change is sound

**Correctness checks:**

1. **Position address lookup** (`PositionCard.tsx:38`): `positionAddress` is computed from `lbPairPosition?.publicKey.toBase58()` which matches how the parent creates the Map key
2. **Data flow**: UPNL fetched at parent â†’ Map by positionAddress â†’ passed to card â†’ card looks up by same key
3. **Type compatibility**: `upnlData?.upnl` and `upnlData?.upnlPercent` are passed to `PositionHeader` which expects `number | null`
4. **Loading state**: `isLoadingPositions || upnlLoading` correctly blocks rendering until data is ready

### âš ď¸Ź Observations (Not Bugs)

1. **Cache key change**: Old `INITIAL_DEPOSITS` cache is abandoned for `UPNL_PER_POSITION`. Users will refetch on update.

2. **Dependency risk**: The change relies on `metcomet@0.2.2` SDK's `getUpnlPerPosition`. If this function has issues, UPNL won't display for all positions. Recommend verifying with a test wallet that has multiple positions.

3. **Deleted code**: 6 files removed (~500 lines). The old implementation had more complex logic including transaction parsing, price fetching at historical timestamps, etc. The new approach is simpler but may have different edge case handling.
