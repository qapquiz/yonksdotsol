# Architecture

Single-source-of-truth reference for system-level patterns. When in doubt, check here first.

## Module Ownership

| Module                 | File                                  | Owns                                           |
| ---------------------- | ------------------------------------- | ---------------------------------------------- |
| Connection singleton   | `src/config/connection.ts`            | The one and only `Connection` instance         |
| Environment            | `src/config/env.ts`                   | All env variable access                        |
| Cache TTL constants    | `src/config/cache.ts`                 | `CACHE_TTL` object                             |
| All caching            | `src/utils/cache/CacheManager.ts`     | Get, set, invalidate, dedup (`getOrFetch`)     |
| Cache key helpers      | `src/utils/cache/cacheKeys.ts`        | Namespaced key generators                      |
| Token data             | `src/tokens/index.ts`                 | Token fetching (via `CacheManager.getOrFetch`) |
| Portfolio calculations | `src/utils/positions/calculations.ts` | Pure functions, no state                       |

## Connection Lifecycle

```
getSharedConnection()  ←  src/config/connection.ts
         │
         ├─→ src/app/index.tsx          (DLMM.getAllLbPairPositionsByUser)
         └─→ src/hooks/useUpnlPerPosition.ts  (metcomet getUpnlPerPosition)
```

- **Single instance** — lazy singleton via module-level `let instance`
- **All consumers import `getSharedConnection()`** — no one creates their own `new Connection()`
- **Never prop-drill Connection** through component trees
- **Never accept Connection as a component/hook prop** — call `getSharedConnection()` where needed

## Caching Strategy

```
CacheManager (singleton)
    │
    ├─→ Token data         key: "token_data:{mint}"        TTL: 60s
    ├─→ UPNL data          key: "upnl_per_position:{addr}" TTL: 15min
    ├─→ OHLCV prices       key: "ohlcv:{pool}:{bucket}"    TTL: 1hr
    └─→ Pyth prices        key: "pyth_price:{sym}:{bucket}" TTL: 1hr
```

- **ALL caching goes through `CacheManager`** — no ad-hoc `Map` caches in modules
- **Request dedup** via `getOrFetch(key, fetchFn, ttl)` — one in-flight promise per key
- **Invalidation** via `delete(key)`, `invalidatePattern(prefix)`, or `clear()`
- Cache keys generated by `src/utils/cache/cacheKeys.ts` helpers — never raw strings

## Data Flow

```
App (index.tsx)
  │  fetches positions via DLMM SDK
  │  manages wallet state + pull-to-refresh
  │  tracks walletResolved (wallet session loaded)
  │  passes Map<PositionInfo> + isLoadingPositions + walletResolved + ownerAddress
  ▼
PositionsList (positions/index.tsx)
  │  aggregates unique mints → useBatchTokenData (batch fetch)
  │  calls useUpnlPerPosition (per wallet)
  │  computes portfolio summary, out-of-range count, position count
  │  renders: PortfolioSummary + PositionCard[]
  ▼
PositionCard
  │  receives: position + lbPositionIndex + upnlData + tokenXInfo + tokenYInfo
  │  computes: totalValue, fees, liquidityShape, priceRange
  │  renders: PositionHeader + LiquidityBarChart + PositionFooter
```

- Token data batched at list level, **never per-card**
- UPNL data fetched per-wallet, looked up per-position via `positionAddress` key
- Position count = `sum(lbPairPositionsData.length)` across all pairs, NOT `positionsArray.length`
